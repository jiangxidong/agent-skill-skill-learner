# Document Templates

Reference structures for each document type. Adapt as needed based on repo content — these are
guides, not rigid forms.

---

## User Guide Template

```markdown
# [Repo Name] User Guide

## Overview
1-2 paragraphs: what this repo is, who made it, what problem it solves, core value proposition.

## Quick Start
### Installation
Step-by-step for the user's specific AI tool.

### Directory Structure
Brief explanation of how skills are organized in this repo.

## Skills Overview

| Skill | Category | Core Function | Use Case |
|-------|----------|---------------|----------|
| ... | ... | ... | ... |

## Skill Details

### [Skill Name]

- **Function**: One-sentence summary
- **Use Cases**: When to trigger this skill
- **Core Value**: What benefit it delivers
- **How to Invoke**: Specific command/phrase for the user's AI tool
- **Prompt Templates**:

Template 1: [scenario description]
\```
[copy-paste prompt]
\```

Template 2: ...

- **Prerequisites**: Required tools, dependencies, or conditions

(Repeat for each skill)

## Recommended Strategy

### By Development Phase

| Phase | Recommended Skills | Purpose |
|-------|-------------------|---------|
| Development | ... | ... |
| Review | ... | ... |
| Release | ... | ... |

### Common Combinations
- **Combo 1**: [name] — [skills involved] → [what it achieves]
- ...
```

## Practice Guide Template

```markdown

## Case Background
- **Project**: Brief description of the fictional project
- **Tech Stack**: Key technologies
- **Scenario**: What situation the developer is in (greenfield, legacy refactor, etc.)

## Development Flow

| Phase | Skills Used | Goal |
|-------|------------|------|
| Phase 1 | ... | ... |
| Phase 2 | ... | ... |
| ... | ... | ... |

## Phase 1: [Phase Name]

### Goal
What this phase achieves.

### Skills Used
- [Skill 1]: why it's used here
- [Skill 2]: why it's used here

### Step 1: [Action]

**Prompt:**
\```
[copy-paste prompt]
\```

**Expected Output:**
\```
[simulated AI response — code snippets, diagnostic reports, etc.]
\```

### Step 2: ...

(Repeat for each phase, ensuring every skill in the repo is used at least once)

## Key Takeaways
- **Lesson 1**: ...
- **Lesson 2**: ...
- **Methodology**: How to think about combining these skills in your own projects
```
